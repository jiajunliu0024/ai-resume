Privacy Policy — Resume Tailor

Effective date: 15 May 2026

This policy describes how the Resume Tailor Chrome extension (“the extension”) handles information. The extension is built to be local-first: it does not require an account, and it does not use a separate application server operated by the extension author to store your résumé or cover letters.

1. Who this policy applies to

You use the extension in Google Chrome (or another compatible Chromium browser). By using the extension, you agree to this policy. If you do not agree, do not use the extension.

2. Data the extension processes

The extension may process the following categories of data on your device or when you choose to use AI features:

  • API credentials you provide. You may enter an API key for an AI provider (for example OpenAI or DeepSeek). The key is stored locally in the browser’s extension storage so you do not have to re-enter it every session.
  • Provider and model-related settings. For example, which provider you selected. Stored locally.
  • Résumé content. Text and structured fields parsed from résumés you upload (including from PDF files), and edits you make inside the extension. Stored locally unless you copy or export it elsewhere yourself.
  • Job description content. Text read from the active browser tab when you run a scan, plus structured fields derived from that text (for example requirements and keywords). Stored locally after a successful scan.
  • Generated outputs. AI-generated suggestions, tailored text, and cover letters produced when you request them. Stored locally as part of normal use of the extension.

3. Where data is stored

Data used by the extension is stored on your device using Chrome’s extension storage APIs (for example chrome.storage.local). The extension does not sync this data to a custom backend operated by the extension author as part of this open-source project.

Uninstalling the extension or clearing extension data through Chrome may remove locally stored data; refer to Google’s documentation for your browser version.

4. Third-party AI providers

When you use features that call an AI model, the extension sends the minimum content needed for that request (for example job text, résumé excerpts, or prompts you trigger) directly from your browser to the third-party provider you configured (for example OpenAI or DeepSeek), using your API key.

Those requests are governed by the privacy policy and terms of that provider, not by this document alone. You should review the provider’s policy before entering a key or sending sensitive information.

The extension’s manifest declares network access only to the provider API hosts required for those calls.

5. Permissions and why they are used

  • activeTab — To read page content from the tab you are using when you take an explicit action (for example scanning a job page).
  • storage — To save your settings, API key, scanned jobs, résumés, and results locally on your device.
  • scripting — To inject or coordinate scripts when needed for the extension UI on web pages (for example a floating panel), in line with Chrome’s security model.
  • Host permissions for AI APIs — To contact the AI provider endpoints you choose when generating or parsing content.

6. Data sharing with the extension author

This project does not include a built-in “phone home” telemetry service. Routine use of the extension does not upload your résumé, cover letter, or API key to a server operated by the extension author solely because you installed the extension.

If you install the extension from the Chrome Web Store, Google may collect usage and crash data according to Google’s privacy policy.

7. Children’s privacy

The extension is not directed at children under 13 (or the age required in your jurisdiction). Do not use it if you are not old enough to consent to processing of your data where you live.

8. Changes to this policy

This policy may be updated when the extension or store requirements change. The “Effective date” at the top will be revised when material changes are made. Continued use after an update means you accept the revised policy.

9. Contact

For questions about this privacy policy or the open-source project, use the issue tracker or contact channel linked from the README of the GitHub repository where you obtained Resume Tailor (update that README with your preferred email or support link for your published build).
